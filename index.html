Why Conversion Is Needed

Claude Code skills have 5 major blockers for Hermes:

Interactive Prompts
input() calls that block automation
External APIs
OpenAI, Firebase requiring custom keys
CLI Dependencies
gh, pip, sudo required
Image Processing
Pillow for icon cropping and BG removal

Three Patterns to Eliminate

❌ DON'T (Claude Code)
  • input("Choose:") — blocks automation
  • subprocess.run(draw.py) — needs OpenAI
  • pip install Pillow — needs install
  • ~/.claude/skills/ — wrong path
VS
✅ DO (Hermes)
  • Pre-configured values — no prompts
  • CSS gradients — self-contained
  • Inline SVGs — no image processing
  • ~/AppData/Local/hermes/skills/ — correct

Conversion Checklist

1. Analyze — Read SKILL.md + install scripts, list all dependencies

2. Replace — Swap each dependency for a Hermes-compatible alternative

3. Restructure — Format SKILL.md with frontmatter, clear workflow, trigger conditions

4. Install — Copy to ~/AppData/Local/hermes/skills/<name>/

5. Verify — Run skill_view(name="<name>") to confirm detection

6. Strip — Remove install.py, PIL scripts, Claude-specific configs
